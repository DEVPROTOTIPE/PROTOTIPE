# 📝 Manual de Contratación de Clientes PROTOTIPE

## 1. Principio general

Todo cliente de PROTOTIPE debe pasar por un proceso de análisis previo antes de iniciar cualquier desarrollo.

No se venden aplicaciones genéricas.

Se diseñan soluciones basadas en el negocio del cliente.

---

## 2. Proceso de contratación

### 2.1 Primer contacto
El cliente puede llegar por:
*   Redes sociales
*   Referidos
*   Publicidad
*   Contacto directo

---

### 2.2 Reunión inicial
Se realiza una reunión gratuita donde se:
*   Escucha el negocio del cliente.
*   Identifican problemas operativos.
*   Entienden procesos actuales.
*   Evalúan necesidades tecnológicas.
*   Analizan branding y operación.

---

### 2.3 Diagnóstico
PROTOTIPE analiza:
*   Complejidad del sistema.
*   Módulos necesarios.
*   Nivel de personalización.
*   Tipo de solución requerida.

---

### 2.4 Propuesta
Se entrega:
*   Alcance del sistema.
*   Funcionalidades.
*   Tiempo estimado.
*   Precio de implementación.
*   Modelo de pago recurrente si aplica.

---

### 2.5 Aprobación
El cliente confirma:
*   Alcance.
*   Precio.
*   Condiciones.
*   Modelo de soporte.

---

### 2.6 Inicio del proyecto
Se realiza:
*   Pago inicial.
*   Diseño del sistema.
*   Desarrollo.
*   Implementación.
*   Capacitación.

---

## 3. Principio clave

El cliente no compra software.

El cliente contrata una solución adaptada a su negocio.
