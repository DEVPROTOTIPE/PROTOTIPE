## 📐 Directivas de Comportamiento Estratégico para la IA

1. **Evaluación de Relevancia Turno Cero:** Antes de abrir cualquier documento, lee la columna de **Criterio de Decisión IA** de este mapa. Si la tarea no coincide con el caso de uso, ignora el archivo para evitar saturar el contexto.
2. **Reutilización Obligatoria:** No crees código nuevo para tareas cubiertas en la Sección 4. Es mandatorio importar/adaptar las plantillas portables.
3. **Persistencia de Documentación (CRÍTICO):** Siempre que agregues, elimines o modifiques cualquier archivo dentro del directorio de documentación (Biblioteca de Componentes, Manuales, Tareas, etc.), debes registrar, actualizar o remover su entrada y Criterio de Decisión en este mapa (`mapa_documentacion_ia.md`) y en el mapa físico (`mapa_aplicacion.md`) de forma obligatoria en el mismo paso del cambio.
4. **Validación de Rutas:** Abre todos los archivos directamente con la URL absoluta proveída en este mapa. No uses búsquedas recursivas si ya tienes la ruta exacta aquí.
5. **Uso Estratégico de Repositorios Externos (CRÍTICO):** Antes de proponer instalar una nueva librería o buscar soluciones en internet, consulta primero el catálogo [`repositorios_github_utiles.md`](file:///D:/PROTOTIPE/Documentacion%20PROTOTIPE/08_Plan_Escalabilidad_Negocio/repositorios_github_utiles.md). Reglas de uso:
   - **NUNCA** copiar código de un repositorio externo directamente. Usarlo como **guía de adaptación** al stack y contexto del proyecto.
   - **SIEMPRE** verificar si la librería ya está instalada en `package.json` antes de proponer instalarla.
   - El catálogo aplica a todos los proyectos del ecosistema: App Ventas, verticales y proyectos de clientes.
   - Si al resolver una funcionalidad se identifica un nuevo repositorio útil no listado, proponer al usuario agregarlo al catálogo.
